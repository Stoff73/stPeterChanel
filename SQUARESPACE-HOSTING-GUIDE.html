Part 1: Before You Begin - What You Need

Before starting, make sure you have the following ready:

Required Items Checklist

  • A computer with internet access (Windows PC or Mac)
  • A modern web browser (Google Chrome is recommended)
  • An email address you can access
  • A payment method (credit/debit card) for Squarespace subscription
  • The website files folder (the folder containing all your HTML, CSS, and JS files)
  • Your school logo image file (PNG or JPG format)
  • Any photos you want to add to the website
  • Your domain name details (if you already own one, e.g., st-peterchanel.bexley.sch.uk)

Part 2: Creating Your Squarespace Account

1Open your web browser

Click on the Google Chrome icon on your computer (or another web browser like Safari or Edge).

2Go to the Squarespace website

In the address bar at the top of your browser (where you type website addresses), type:

www.squarespace.com

Then press the Enter key on your keyboard.

3Click "Get Started"

On the Squarespace homepage, look for a button that says Get Started. It's usually in the top right corner of the page. Click on it.

4Create your account

You'll be asked to create an account. You have several options:

  • Sign up with Email: Enter your email address, then create a password
  • Sign up with Google: If you have a Gmail account, you can use this
  • Sign up with Apple: If you have an Apple ID, you can use this

We recommend using your school email address if possible.

5Verify your email

Squarespace will send a verification email to the address you provided. Go to your email inbox, find the email from Squarespace, and click the verification link inside.

If you don't see the email, check your "Spam" or "Junk" folder. The email might take a few minutes to arrive.

Part 3: Choosing and Setting Up Your Plan

1Skip the template selection (for now)

After creating your account, Squarespace will show you templates to choose from. For our purposes, we'll select the simplest template possible because we'll be replacing most of it with our own code.

Choose the template called "Blank" or the simplest option available (often called "Brine" or similar).

2Enter your site name

When prompted, enter your school name:

St Peter Chanel Catholic Primary School

3Start your trial

Squarespace offers a 14-day free trial. Click Start Trial to begin. You can explore during the trial before paying.

4Upgrade to Business Plan

Before we can add our custom code, you need to upgrade to the Business plan:

  1. Look at the left side of the screen for a menu
  2. Click on Settings (it has a gear icon)
  3. Click on Site Availability
  4. Click on Upgrade or Subscribe
  5. Select the Business plan
  6. Choose billing frequency (Annual is cheaper per month)
  7. Enter your payment details
  8. Complete the purchase
You must complete the upgrade before proceeding. The Code Injection feature we need is only available on Business plans and above.

Part 6: Adding the CSS Styles

The CSS file (styles.css) controls how your website looks. We need to add it to Squarespace.

Option A: Using Custom CSS (Recommended)

1Open the Custom CSS area

From your Squarespace dashboard:

  1. Click on Design in the left menu
  2. Click on Custom CSS

A panel will open on the left side of the screen with a large text box.

2Open your styles.css file

On your computer:

  1. Find your website folder
  2. Find the file called styles.css
  3. Right-click on the file
  4. Select "Open with" ‚Üí "Notepad" (on Windows) or "TextEdit" (on Mac)

The file will open showing lots of code.

3Copy all the CSS code

  1. Click anywhere in the text
  2. Press Ctrl + A (Windows) or Cmd + A (Mac) to select all the text
  3. Press Ctrl + C (Windows) or Cmd + C (Mac) to copy

4Paste into Squarespace

  1. Go back to your Squarespace Custom CSS panel
  2. Click inside the large text box
  3. Press Ctrl + V (Windows) or Cmd + V (Mac) to paste

5Save your changes

Click the Save button at the top of the panel.

After saving, you should see some of your styles appear on the page preview. If something looks wrong, don't worry - we still need to add the HTML and JavaScript.

Part 7: Adding the JavaScript

The JavaScript file (script.js) makes interactive features work, like the mobile menu and cookie consent.

1Go to Code Injection

  1. Click on Settings in the left menu
  2. Click on Developer Tools
  3. Click on Code Injection

2Open your script.js file

On your computer:

  1. Find your website folder
  2. Find the file called script.js
  3. Right-click on the file
  4. Select "Open with" ‚Üí "Notepad" (on Windows) or "TextEdit" (on Mac)

3Copy all the JavaScript code

  1. Click anywhere in the text
  2. Press Ctrl + A to select all
  3. Press Ctrl + C to copy

4Add to Footer section

In the Code Injection page, scroll down to find the section labelled "Footer".

In the Footer text box, type the following exactly:

<script> PASTE YOUR JAVASCRIPT CODE HERE </script>

Replace "PASTE YOUR JAVASCRIPT CODE HERE" with your copied code by pressing Ctrl + V.

5Add Google Translate (also in Footer)

After the closing </script> tag, add these two lines:

<script> function googleTranslateElementInit(){ new google.translate.TranslateElement({pageLanguage:'en'},'google-translate'); } </script> <script src="//translate.google.com/translate_a/element.js?cb=googleTranslateElementInit"></script>

6Add Font Awesome (in Header section)

Scroll up to the "Header" section and add:

<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.4.0/css/all.min.css"> <link rel="preconnect" href="https://fonts.googleapis.com"> <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin> <link href="https://fonts.googleapis.com/css2?family=Open+Sans:wght@300;400;600;700&family=Montserrat:wght@400;500;600;700&display=swap" rel="stylesheet">

7Save your changes

Click the Save button at the top of the page.

Part 8: Creating Pages and Adding HTML Content

Now we need to create each page and add the HTML content. This is the most time-consuming part.

Creating the Homepage

1Go to Pages

In the left menu, click on Pages.

2Edit the existing homepage

You should see a page called "Home" or similar. Hover over it and click Edit.

3Add a Code Block

  1. When editing the page, click the + button to add a block
  2. In the menu that appears, look for "Code" or search for it
  3. Click on the Code block to add it

4Open your index.html file

On your computer, open index.html with Notepad or TextEdit.

5Copy the content section

You only need to copy the part between <body> and </body> tags, but NOT including the script tags at the bottom.

Look for everything from after <body> up to (but not including) <script src="script.js">.

6Paste into the Code Block

  1. In the Code Block dialog box, make sure "Display Source" is unchecked
  2. Paste your HTML code into the text area
  3. Click Apply

7Save the page

Click Save or Done in the top corner.

Creating Additional Pages

Repeat this process for each page on your website. For each page:

1Create a new page

  1. Go to Pages in the left menu
  2. Click the + button
  3. Select Blank Page
  4. Give it a name (e.g., "About Us")

2Add a Code Block and paste HTML

Follow the same steps as for the homepage:

  1. Edit the page
  2. Add a Code Block
  3. Open the corresponding HTML file (e.g., about-us.html)
  4. Copy the body content (excluding scripts)
  5. Paste into the Code Block
  6. Save

Part 9: Setting Up Navigation Menus

Since your HTML already contains navigation menus, they should appear automatically. However, you may need to disable Squarespace's built-in navigation:

1Access site styles

  1. Click on Design in the left menu
  2. Click on Site Styles

2Hide the default header

Look for options related to:

  • Header (set visibility to hidden if possible)
  • Navigation (disable if possible)
  • Logo (hide if using your own)

The exact options vary by template. You may need to explore the settings.

3Hide the default footer

Similarly, look for footer options and hide the default footer since your HTML includes a custom footer.

If you can't completely hide the default navigation, you can add CSS to the Custom CSS area to hide it:
header#header { display: none !important; } footer.Footer { display: none !important; }

Part 10: Connecting Your Domain Name

If you already own a domain name (like st-peterchanel.bexley.sch.uk), you can connect it to your Squarespace site.

Option A: Transfer Domain to Squarespace

1Go to Domains settings

  1. Click on Settings
  2. Click on Domains

2Add a domain

Click Use a Domain I Own

3Enter your domain

Type your domain name and follow the instructions to verify ownership.

Option B: Point Domain to Squarespace (Keep at Current Provider)

If you want to keep your domain with your current provider (like your IT department manages it), you'll need to update DNS records:

1Get Squarespace DNS settings

In Domains settings, click to connect a domain and select "Connect a Domain from Another Provider". Squarespace will show you the records you need to add.

2Contact your domain provider/IT department

Provide them with the following information (from Squarespace):

  • CNAME record pointing to verify.squarespace.com
  • A records pointing to Squarespace IP addresses
DNS changes can take 24-48 hours to take effect worldwide. Don't panic if your site doesn't immediately appear on your domain.

Option C: Use a Squarespace Domain (Temporary or Permanent)

You can also purchase a new domain through Squarespace, or use the free .squarespace.com subdomain they provide while you set things up.

Part 11: Adding Your Logo and Images

Uploading Images

1Access the asset library

When editing any page, you can upload images. Or go to:

  1. Click on Design
  2. Look for Custom Files or similar

2Upload your images

  1. Click Upload or drag and drop files
  2. Wait for upload to complete
  3. Note the URL of each image (you'll need this)

Adding Images to Your Pages

Currently, your HTML has placeholder divs for images. To add real images:

1Find the placeholder in your HTML

Look for code like:

<div class="logo-placeholder"></div>

2Replace with an image tag

Replace the placeholder with:

<img src="YOUR_IMAGE_URL" alt="Description of image">

Replace YOUR_IMAGE_URL with the URL from Squarespace's asset library.

3Update in Squarespace

  1. Edit the page in Squarespace
  2. Click on the Code Block
  3. Find and update the HTML
  4. Save changes
Keep image file sizes small (under 500KB) for faster loading. Use JPG for photos and PNG for logos with transparent backgrounds.

2Check every page

Click through every link and page to ensure:

  • Text is readable
  • Images appear correctly
  • Links go to the right places

3Test interactive features

  • Test the cookie consent popup
  • Try the high contrast button
  • Test the Google Translate dropdown

Mobile Testing

1Use device preview

In the Squarespace editor, look for a device icon that lets you preview on different screen sizes (phone, tablet, desktop).

2Test on a real phone

Open your site's preview URL on your mobile phone to test:

  • Text is large enough to read
  • Buttons are easy to tap
  • Mobile menu opens and closes properly
  • Pages scroll smoothly

Testing Checklist

  • Homepage loads correctly
  • All main navigation links work
  • All subpage links work
  • Mobile menu opens and closes
  • Cookie consent appears and buttons work
  • High contrast mode toggles on and off
  • Google Translate widget appears

  • Forms can be filled out and submitted

Troubleshooting Common Problems

Problem: The website looks wrong or unstyled

Possible causes and solutions:

  • CSS not added: Check that you've added all the CSS to the Custom CSS area
  • CSS has errors: Check you copied the entire file without missing any characters
  • Browser cache: Press Ctrl+Shift+R to force refresh the page

Problem: JavaScript features don't work (menu, cookies, etc.)

Possible causes and solutions:

  • Script not added: Check the Code Injection Footer section has the script
  • Script tags wrong: Ensure code is wrapped in <script> tags
  • Syntax error: Check browser console for errors (press F12)

Problem: Links don't work or go to wrong pages

Possible causes and solutions:

  • Page doesn't exist: Create the missing page in Squarespace
  • URL slug wrong: Check the page's URL slug matches the link
  • Link path wrong: Update the href in the HTML to match Squarespace URLs

Problem: Images don't appear

Possible causes and solutions:

  • Placeholder divs: Replace placeholders with actual <img> tags
  • Wrong URL: Get the correct URL from Squarespace's asset library
  • File not uploaded: Upload the image to Squarespace first

Problem: Mobile menu doesn't open

Possible causes and solutions:

  • JavaScript error: Check the console for errors (F12 in browser)
  • ID mismatch: Ensure HTML element IDs match those in the JavaScript

Problem: "Code Injection" option is missing

Solution:

You need to upgrade to the Business plan or higher. The Personal plan doesn't include Code Injection.

Problem: Cookie consent keeps appearing

Possible causes:

  • Browser blocking cookies: Check browser privacy settings
  • Incognito mode: Cookies don't persist in private browsing
  • Script error: Check the JavaScript for errors
